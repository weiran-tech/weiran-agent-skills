# Testany 故障诊断

分析 Testany 测试失败原因，排查问题根因。

## 使用方式

$ARGUMENTS

## 支持的操作

- **分析执行**：获取失败执行的详情
- **查看日志**：获取并分析执行日志
- **定位问题**：识别失败类型和原因
- **修复建议**：提供针对性的修复方向

## 示例

```
/debug Y2K-0601-00001
/debug 最近的执行为什么失败了
/debug 帮我看看日志
```
