# PRD Writer

启动 PRD 撰写流程。帮助你撰写清晰、完整、可执行的产品需求文档。

## 使用方式

描述你想要的功能或需求：

$ARGUMENTS

## 支持的 PRD 类型

1. **新功能（有 UI）** - 涉及用户界面的新功能
2. **新功能（无 UI / 后端）** - 后端服务、API、后台任务
3. **第三方集成** - 接入外部服务
4. **功能重构** - 不改变外部功能的内部重构
5. **性能/安全优化** - 非功能性改进

## 工作流程

1. **上下文收集**：扫描项目文档，了解现有约定
2. **需求理解**：通过提问澄清需求
3. **结构规划**：选择合适的模板
4. **内容撰写**：按模板填充内容
5. **强制审查**：完整性、一致性、边界检查

如果没有提供具体需求，请告诉我你想实现什么功能。
