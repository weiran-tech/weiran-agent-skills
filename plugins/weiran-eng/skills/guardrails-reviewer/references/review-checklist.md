# Guardrails 审查清单

## Gate 1：元信息与范围（P0/P1）
- 版本/Owner/状态/生效时间/复审周期齐全（缺失 → P0）
- 范围与非范围明确（缺失 → P0）
- 更新机制与触发条件明确（缺失 → P1）

## Gate 2：覆盖性（P0/P1）
- 关键领域覆盖（安全、API、数据、发布、可观测性）（缺失 → P0）
- LLD 模块要求表完整（缺失 → P0）
- 前端/后端/运维至少一项规则（缺失 → P1）

## Gate 3：可执行性（P0/P1）
- 每条规则有 Level 与 Verification（缺失 → P0）
- 规则可验证且可落地（不可执行 → P1）
- 规则不冲突、不重复（冲突 → P1）

## Gate 4：一致性（P1/P2）
- 与现有技术栈/规范不冲突（冲突 → P1）
- 例外流程清晰、可执行（缺失 → P1）
- 不包含实现细节（发现 → P2）
